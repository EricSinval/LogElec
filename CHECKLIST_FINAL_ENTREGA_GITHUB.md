# Checklist Final de Entrega no GitHub

Use esta checklist antes de apresentar e antes de considerar a branch de entrega como fechada.

## Código

- [ ] `README.md` revisado e coerente com a primeira entrega
- [ ] interface de Login funcionando
- [ ] interface Esqueci a Senha funcionando
- [ ] cadastro de usuário funcionando
- [ ] leitura dos dados do usuário funcionando
- [ ] atualização dos dados do usuário funcionando
- [ ] exclusão de conta funcionando ou, se bloqueada por vínculo, regra explicada

## Validação técnica

- [ ] `docker compose up --build -d` executa sem erro
- [ ] frontend abre corretamente
- [ ] backend responde em `http://localhost:8080`
- [ ] fluxo de login testado
- [ ] fluxo de recuperação de senha testado
- [ ] fluxo de CRUD de usuário testado

## GitHub

- [ ] branch de entrega criada com nome válido no Git
- [ ] commit final realizado
- [ ] push executado para o GitHub
- [ ] repositório remoto atualizado com o último commit
- [ ] arquivos sensíveis locais não foram enviados por engano

## Apresentação em sala

- [ ] começar mostrando o objetivo da primeira entrega
- [ ] apresentar primeiro o cadastro do usuário
- [ ] depois demonstrar login
- [ ] depois demonstrar perfil como leitura, edição e exclusão
- [ ] finalizar com Esqueci a Senha
- [ ] citar módulos extras só se houver tempo ou se o professor perguntar

## Entrega

- [ ] enviar somente o projeto em código, como solicitado pelo professor
- [ ] confirmar que o link do GitHub abre corretamente
- [ ] confirmar que a branch apresentada é a branch correta da entrega
